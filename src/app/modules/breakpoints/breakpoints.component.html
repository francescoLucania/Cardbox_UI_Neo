breakpoints - упрощенная работа с @media выражениями

Миксины

Для системной и упрощенной работы с @media в системе есть миксин mq() и mq-custom()

Миксин mq()

В файле _variables.scss есть $mq-breakpoints - "массив" с данными по брекпоинтам(на основе Cardbox UI), в нем указаны системные брекпоинты портала.

mq() работает на основе данных в $mq-breakpoints, первый ключ - sm, {{'mq(sm) {...}'}} сгенерирует следующий css: {{'@media all and (max-width: 767px) {...}'}}

max-width генерируется только по первому ключу массива(в нашем случае это sm)

{{'mq(md) {...}'}} сгенерирует следующий css: {{'@media all and (min-width: 768px) {...}'}}

{{'mq(lg) {...}'}} сгенерирует следующий css: {{'@media all and (min-width: 1140px) {...}'}}

Миксин mq-custom()

mq-custom() помогает делать кастомные брекпоинты(не отраженные в $mq-breakpoints), у него два атрибута: mq-custom( max/min, 640px )

{{'@include mq-custom (max, 640px) {...}'}} сгенерирует следующий css: {{'@media all and (max-width: 640px) {...}'}}

{{'@include mq-custom (min, 720px) {...}'}} сгенерирует следующий css: {{'@media all and (min-width: 720px) {...}'}}
